# Day 42: AI 安全與隱私 (AI Security & Privacy)

## 0. 前言：AI 也是會被駭的
隨著 AI 應用越來越廣，它也成為了駭客的目標。
*   **攻擊者**想騙過你的模型 (例如讓自駕車看不見停止標誌)。
*   **攻擊者**想偷你的訓練資料 (例如還原出病人的病歷)。

本章節整理了 IPAS 考題中關於 **對抗式攻擊、隱私計算、倫理法規** 的關鍵知識。

## 1. 對抗式攻擊 (Adversarial Attacks)
這是一種「針對 AI 的光學幻覺」。

### 1.1 原理
*   在圖片上加入人類肉眼看不見的微小雜訊 (Perturbation)。
*   這些雜訊是經過精心計算的 (沿著梯度的反方向)，專門用來誤導神經網路。
*   **結果**：人眼看是「熊貓」，AI 卻以 99% 信心說是「長臂猿」。(考題 Q24)

### 1.2 防禦策略
*   **對抗式訓練 (Adversarial Training)**：把這些攻擊樣本加入訓練集，讓模型「看過世面」，學會抵抗。
*   **蒸餾 (Distillation)**：用一個大模型教小模型，隱藏梯度資訊。

## 2. 隱私計算 (Privacy-Preserving Computation)
如何在 **不洩漏個資** 的情況下訓練 AI？這是醫療與金融業的聖杯。

### 2.1 同態加密 (Homomorphic Encryption)
*   **定義**：一種神奇的加密技術，允許你在 **加密狀態下直接進行數學運算**。(考題 Q36)
*   **流程**：
    1.  數據持有者將 $x$ 加密成 $E(x)$。
    2.  雲端伺服器計算 $E(x) + E(y)$，得到 $E(x+y)$ (它完全不知道 $x, y$ 是什麼)。
    3.  數據持有者解密得到 $x+y$。
*   **優點**：絕對隱私。
*   **缺點**：運算速度極慢 (比明文慢 100~1000 倍)。

### 2.2 安全多方計算 (MPC - Secure Multi-Party Computation)
*   **定義**：多個人共同計算一個函數，但沒人知道其他人的輸入。(考題 Q37)
*   **例子 (百萬富翁問題)**：兩個富翁想比誰更有錢，但不想透露自己具體有多少錢。
*   **技術組合**：通常結合了 同態加密、雜湊函數、對稱加密、秘密分享 (Secret Sharing)。

### 2.3 聯邦學習 (Federated Learning)
*   **口訣**：**數據不動，模型動**。
*   **流程**：
    1.  Google 把模型寄到你的手機裡。
    2.  你的手機用你的照片訓練模型 (更新權重)。
    3.  手機只把 **更新後的權重 (Gradients)** 寄回 Google，**照片留在手機裡**。
*   **應用**：Gboard 輸入法預測、醫療影像聯邦訓練。

## 3. AI 倫理與法規
### 3.1 偏見 (Bias)
*   **來源**：
    *   **標籤偏差**：標註人員的主觀偏見。(考題 Q13)
    *   **取樣偏差**：訓練資料只包含特定族群 (如只收集活躍用戶)。(考題 Q28)
    *   **跨文化誤差**：詞嵌入 (Word Embedding) 在不同文化下的語意不同。(考題 Q49)
*   **解法**：數據平衡、去偏見演算法 (Debiasing)。

### 3.2 智慧財產權與不可否認性
*   **生成式 AI 侵權**：AI 畫的圖算誰的？有沒有抄襲？
    *   **預防**：建立訓練資料的授權驗證機制。(考題 Q25)
*   **不可否認性 (Non-repudiation)**：
    *   如何證明這筆交易是你做的？這個模型是你訓練的？
    *   **解法**：數位簽章 (Digital Signature) + 雜湊 (Hash)。(考題 Q30)

## 4. IPAS 考題詳解 (安全與隱私篇)
| 題目 (關鍵字) | 答案 | 解析 |
| :--- | :--- | :--- |
| **13. 標籤偏差原因** | (B) 標記資料帶有主觀偏見 | 人工標註必然包含人的主觀意識 (Subjectivity)。 |
| **24. 對抗性攻擊防禦 (非)** | (D) 強化防火牆 | 防火牆擋的是網路封包，擋不住「長得像熊貓的惡意圖片」。 |
| **25. 生成內容侵權預防** | (B) 建立訓練資料篩選與授權驗證 | 源頭管理 (Data Governance) 是最根本的解法。 |
| **28. 取樣偏差** | (C) 訓練樣本僅涵蓋高活躍顧客 | 樣本不具代表性 (Not Representative)，導致模型以偏概全。 |
| **30. 不可否認性** | (A) 記錄雜湊值並簽署數位簽章 | 這是資安領域確保「原創性」與「完整性」的標準做法。 |
| **36. 同態加密特性** | (D) 加密狀態下仍可進行運算 | 這是同態加密(HE)最獨特的定義。 |
| **37. 安全多方計算** | (D) 同態加密 + MPC + 雜湊 + 對稱加密 | MPC 是一個技術集合體，通常混合多種加密技術。 |
| **49. 跨文化情感分析 (非)** | (A) 詞嵌入正規化導致誤差 | 正規化是數學操作，通常不會導致文化誤差；誤差來自訓練語料的文化單一性。 |

## 5. 總結
AI 安全是一個「矛與盾」的對決。
*   我們用 **對抗式訓練** 來防禦攻擊。
*   我們用 **同態加密/聯邦學習** 來保護隱私。
*   我們用 **去偏見技術** 來確保公平。
